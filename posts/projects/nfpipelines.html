Building workflows is where everything started. My first pipeline was insertseq, I started the development of this during my master's thesis, and we ended up publishing. The pipeline performs the analysis of insertional mutagenesis experiments.

Later, we developed nf-core/crisprseq, which I am author and maintainer of. This pipeline performs the analysis of almost all CRISPR-Cas experiments (targeted and screening) and is used by many researchers around the world.

I also contribute to several other nf-core pipelines by reviewing code, improving modules, and helping ensure compatibility with evolving nf-core and Nextflow standards.

⚙️ Some technical aspects:

  • Pipelines built using Nextflow DSL2
  • They are modular, using shared nf-core modules and subworkflows
  • nf-core pipelines are tested with GitHub Actions
